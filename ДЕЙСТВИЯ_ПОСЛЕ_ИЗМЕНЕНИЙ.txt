КРАТКАЯ ВЕРСИЯ:

1️⃣  УЗНАЙТЕ IP WINDOWS МАШИНЫ
    
    На Windows (в PowerShell/CMD, НЕ в контейнере):
    
    ipconfig
    
    Найдите IPv4 адрес вида: 192.168.1.XXX


2️⃣  НАСТРОЙТЕ ПРОБРОС ПОРТОВ В DOCKER

    Вариант А: В docker-compose.yml добавьте:
    
    ports:
      - "3005:3005"
      - "8005:8005"
    
    Затем: docker-compose down && docker-compose up -d
    
    Вариант Б: При запуске docker run:
    
    docker run -p 3005:3005 -p 8005:8005 ...
    
    
3️⃣  ПРОВЕРЬТЕ ПРОБРОС ПОРТОВ

    docker ps
    
    Должно быть: 0.0.0.0:3005->3005/tcp, 0.0.0.0:8005->8005/tcp


4️⃣  НАСТРОЙТЕ WINDOWS FIREWALL

    PowerShell с правами администратора:
    
    New-NetFirewallRule -DisplayName "Django" -Direction Inbound -LocalPort 8005 -Protocol TCP -Action Allow
    New-NetFirewallRule -DisplayName "React" -Direction Inbound -LocalPort 3005 -Protocol TCP -Action Allow


5️⃣  ОБНОВИТЕ IP В КОНФИГУРАЦИИ

    В контейнере:
    
    ./setup_ip_for_ghpages.sh
    
    Введите IP Windows из шага 1 (например: 192.168.1.100)


6️⃣  ЗАПУСТИТЕ ПРИЛОЖЕНИЕ

    ./run_backend.sh    # Терминал 1
    ./run_frontend.sh   # Терминал 2


7️⃣  ПРОВЕРЬТЕ ДОСТУП

    С Windows:
    • http://localhost:3005
    • http://localhost:8005
    
    С телефона (в той же WiFi):
    • http://192.168.1.XXX:3005
    • http://192.168.1.XXX:8005

│  Проблема: Docker контейнер имеет СВОЙ IP (172.18.0.X)                      │
│  Это НЕ тот IP, который нужен для доступа извне!                            │
│                                                                              │
│  Телефон → WiFi сеть → Windows IP (192.168.1.X)                             │
│                            ↓                                                 │
│                       Docker Desktop                                         │
│                            ↓                                                 │
│                     Контейнер (172.18.0.X)                                   │
│                                                                              │
│  Решение: Проброс портов (port mapping) + правильный IP в конфиге           │
└──────────────────────────────────────────────────────────────────────────────┘
